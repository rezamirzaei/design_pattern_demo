üí° How Command Works Here:
  • Each action creates a Command object (TurnOnCommand, TurnOffCommand, etc.)
  • Commands have execute() and undo() methods
  • An Invoker stores command history for undo/redo
  • Commands can be queued, logged, and even recorded as macros
  • Device (Receiver) doesn't know about Command pattern - loose coupling!
